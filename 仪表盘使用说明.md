# 会议改善效果评估仪表盘 - 使用说明

生成时间: 2025-11-20

---

## 📊 仪表盘概述

本仪表盘完全按照《可视化报表页面设计方案.md》实现,提供四大核心模块:

1. **🏠 概览页面** - 核心KPI、趋势分析、快速洞察
2. **📊 原始数据页面** - 数据筛选、表格展示、个人详情
3. **📈 分析结果页面** - KPI详细分析、统计对比、类型分布
4. **👥 人员详情页面** - Top10用户、分层统计、异常检测

---

## 🚀 快速开始

### 1. 生成仪表盘

```bash
# 运行生成脚本
python3 generate_full_dashboard.py

# 输出文件位置
# output/meeting_dashboard_full.html
```

### 2. 打开仪表盘

```bash
# 在浏览器中打开
open output/meeting_dashboard_full.html

# 或者双击HTML文件
```

### 3. 数据来源

仪表盘自动加载 `input/` 目录下的所有CSV数据文件:
- 9月会议详情.csv (基线期)
- 10月会议详情.csv (基线期)
- 10.20-10.26会议详情.csv (周度数据)
- 10.27-11.2会议详情.csv (周度数据)
- 11.03-11.09会议详情.csv (周度数据)
- 11.10-11.16会议详情.csv (周度数据)

## 📋 功能详解

### 模块1: 🏠 概览页面

#### 核心KPI卡片
- **显示内容**: 4个主要KPI的达标情况
- **交互**: 点击卡片可跳转到详细分析页面
- **状态标识**:
  - 🟢 达标
  - 🔴 未达标
  - 🟡 需关注
  - 🟠 接近达标

#### 趋势分析图
- **类型**: 折线图
- **内容**: 日人均会议数的周期变化趋势
- **功能**: 鼠标悬停查看具体数值

#### 会议类型分布
- **类型**: 柱状对比图
- **内容**: 基线期vs当前期的会议类型对比
- **包含**: 即时会议、日程会议、1v1通话

#### 快速洞察卡片
- Top10用户统计
- 异常检测结果
- 团队均衡度(CV值)
- 高频用户数量

**交互**: 点击卡片跳转到对应详情页面

---

### 模块2: 📊 原始数据页面

#### 数据筛选器
- **时间段筛选**: 下拉选择周期
- **人员搜索**: 输入姓名模糊搜索
- **操作按钮**:
  - 应用筛选
  - 重置筛选

#### 数据表格
- **列**: 姓名、周期、日均会议数、日均时长、即时会议、日程会议、1v1通话
- **功能**:
  - 点击列标题排序
  - 分页显示(每页50条)
  - 点击"详情"按钮查看个人详情

#### 数据导出
- **CSV导出**: 导出筛选后的数据为CSV文件
- **JSON导出**: 导出筛选后的数据为JSON文件

#### 个人详情弹窗
点击"详情"按钮后显示:
- 基本信息(排名、变化率)
- 全部周期数据表格
- 基线期vs当前期对比

---

### 模块3: 📈 分析结果页面

#### KPI详细分析
每个KPI展开显示:
- 基线期均值
- 当前期均值
- 变化率
- 达标状态
- 评估说明

#### 描述性统计对比
- **雷达图**: 展示基线期vs当前期的5个统计指标
  - 均值
  - 标准差
  - 变异系数
  - 最大值
  - 最小值

#### 会议类型详细分析
- **双饼图**: 基线期和当前期的类型分布对比
- **数据表**: 显示各类型的数量、占比和趋势

---

### 模块4: 👥 人员详情页面

#### Top10用户卡片
- **显示**: 重度会议用户Top10
- **内容**:
  - 排名
  - 姓名
  - 基线期会议数
  - 当前期会议数
  - 变化率
  - 改善/增加状态
- **交互**: 点击卡片查看个人详情弹窗

#### 人员分层统计
- **柱状图**: 显示高频、中频、低频用户分布
- **分类标准**:
  - 高频: ≥5次/天
  - 中频: 2-5次/天
  - 低频: <2次/天

#### 异常检测列表
- **表格显示**: 所有异常用户
- **字段**:
  - 序号
  - 姓名
  - 异常指标
  - 当前值
  - 整体均值
  - Z-score
  - 异常类型(高于/低于平均)

## 🎨 设计特性

### 视觉设计
- **渐变背景**: 紫色渐变背景
- **卡片样式**: 圆角白色卡片,阴影效果
- **颜色标识**:
  - 绿色 - 良好/改善
  - 红色 - 警告/增加

- **标签页切换**: 平滑过渡动画

- **点击反馈**: 视觉反馈
- **弹窗**: 模态窗口展示详情

### 响应式设计
- **桌面端**: 多列网格布局
- **平板端**: 2列布局
- **移动端**: 单列垂直布局

## 🔄 更新数据

### 1. 添加新数据
```bash
# 将新的CSV文件放入input目录
cp 新周期会议详情.csv input/

# 重新生成仪表盘
python3 generate_full_dashboard.py
```

### 2. 数据格式要求
CSV文件必须包含以下字段:
- user_name: 用户姓名
- period_name: 周期名称
- 日人均线上会议数: 数值
- 日人均线上会议时长(分钟): 数值
- 即时会议数: 整数
- 日程会议数: 整数
- 1v1通话数: 整数

## 🐛 常见问题

### Q1: 仪表盘显示空白
**A**: 检查浏览器控制台是否有错误,确保ECharts CDN可访问

### Q2: 数据没有更新
**A**: 重新运行 `python3 generate_full_dashboard.py`

### Q3: 图表不显示
**A**: 等待ECharts加载完成,或刷新页面

### Q4: 筛选不生效
**A**: 点击"应用筛选"按钮,而不是直接改变下拉框

### Q5: 导出功能无响应
**A**: 检查浏览器是否允许下载文件
