## 📊 核心指标说明

### 主要KPI（按优先级）

1. **日人均线上会议数（即时+日程）** ⭐
   - 定义: 平均每人每天参加的线上会议次数（不含1v1通话）
   - 目标: 减少会议数量
   - 健康范围: 2-5次/天

2. **日人均线上会议时长（即时+日程）** ⭐
   - 定义: 平均每人每天在线上会议中的总时长（分钟）
   - 目标: 减少会议时长
   - 健康范围: 60-180分钟/天

3. **即时会议占比** ⭐
   - 定义: 即时会议数 ÷ 总会议数 × 100%
   - 目标: 降低即时会议占比，提高会议计划性
   - 健康范围: <30%

### 次要指标

4. **会议类型分布**
   - 即时会议、日程会议、1v1通话的比例

5. **用户分层**
   - 高频用户: ≥5次/天
   - 中频用户: 2-5次/天
   - 低频用户: <2次/天

## 🎯 常见分析场景

### 场景1: 周度例行检查（每周一）

**目的**: 监测上周会议情况

**操作步骤**:
1. 上传上周CSV（如11.17-11.24）
2. 基线期: 9月+10月
3. 当前期: 最近4周
4. 查看概览页面的KPI变化
5. 关注Top10用户是否有变动

**关注重点**:
- KPI是否在目标范围内
- 是否有新的高频用户
- 即时会议占比趋势

---

### 场景2: 月度总结（每月初）

**目的**: 生成月度分析报告

**操作步骤**:
1. 确保上传了整月的周度数据
2. 导出整月汇总CSV
3. 基线期: 9月
4. 当前期: 上个月
5. 导出"原始数据"页面的数据

**输出内容**:
- 月度KPI对比
- Top用户排名变化
- 用户分层趋势
- 主责人员表现

---

### 场景3: 个人深度分析

**目的**: 分析特定用户的会议习惯

**操作步骤**:
1. 进入"原始数据"页面
2. 搜索框输入用户名
3. 点击"查看详情"
4. 或进入"人员详情"页面选择用户

**查看内容**:
- 该用户的历史趋势
- 每周会议数变化
- 会议类型分布
- 与平均水平对比

## 📈 推荐的周期选择策略

### 策略1: 固定基线（推荐）✅
```
基线期: 始终选择 9月 + 10月
当前期: 滚动更新（最近N周）
```

**优点**:
- 对比基准一致
- 趋势清晰
- 容易看出改善效果

**适用场景**: 日常监测

---

### 策略2: 滚动对比
```
基线期: 上个月
当前期: 本月
```

**优点**:
- 及时发现变化
- 敏感度高

**适用场景**: 月度报告

---

### 策略3: 长短期对比
```
基线期: 最初2个月（9月+10月）
当前期: 所有后续周期
```

**优点**:
- 全面评估
- 看长期趋势

**适用场景**: 季度/年度总结

## 🔧 常见问题

### Q1: KPI卡片显示0或空值？
**原因**: 未选择基线期或当前期
**解决**: 在"数据管理"页面选择周期并点击"加载数据"

### Q2: 趋势图显示乱序？
**状态**: ✅ 已修复（2025-11-24）
**效果**: 现在按月份从小到大排序

### Q3: 找不到某个用户？
**方法**:
- 在"原始数据"页面使用搜索框
- 或在"人员详情"页面下拉选择

### Q4: 如何查看主责人员统计？
**前提**: 需要先配置主责人员
**方法**:
- 之前在"配置"页面添加
- 现在直接修改 `config/responsible_persons.json`

### Q5: 数据太多，加载很慢？
**优化**:
- 在"原始数据"页面先筛选周期
- 减少一次加载的数据量
- 考虑按月而非按周上传
