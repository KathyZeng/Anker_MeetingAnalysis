# 会议分析可视化报表 - 页面设计方案

## 🎨 设计理念

**双层架构**: 原始数据展示 + 统计分析结果
**交互特性**: 可点击卡片跳转到详细数据
**响应式设计**: 支持桌面和移动端访问

## 🎨 交互设计细节

### 交互 1: 点击KPI卡片
```
用户操作: 点击"日人均会议数减少率"卡片
系统响应:
  1. 卡片展开显示详细统计数据
  2. 显示受影响人员列表
  3. 显示周度趋势图
  4. 提供"查看原始数据"链接
```

### 交互 2: 点击Top10用户卡片
```
用户操作: 点击"Sam Liu"用户卡片
系统响应:
  1. 弹出模态窗显示用户详情
  2. 加载该用户的原始数据(所有周期)
  3. 显示个人会议数趋势图
  4. 显示会议类型分布
  5. 提供"导出该用户数据"功能
```

### 交互 3: 数据表格筛选
```
用户操作: 在原始数据页选择"10月"周期
系统响应:
  1. 表格自动筛选显示10月数据
  2. 更新统计卡片显示该周期统计
  3. 高亮显示筛选条件
  4. 提供"清除筛选"按钮
```

### 交互 4: 异常用户详情
```
用户操作: 点击异常用户列表中的"详情"
系统响应:
  1. 跳转到该用户的个人详情页
  2. 高亮显示异常指标
  3. 显示异常趋势对比
  4. 提供改善建议
```

## 🎯 技术实现要点

### 前端技术栈
- **框架**: HTML5 + CSS3 + JavaScript (原生或Vue.js)
- **图表库**: ECharts / Chart.js
- **表格组件**: DataTables / AG-Grid
- **UI框架**: Bootstrap 5 / Tailwind CSS
- **交互**: AJAX异步加载数据

### 数据流
```
原始CSV数据
    ↓
Python数据处理
    ↓
生成JSON数据文件
    ↓
HTML页面通过AJAX加载
    ↓
ECharts/表格组件渲染
    ↓
用户交互(点击/筛选)
    ↓
动态更新显示内容
```

### 文件结构
```
output/
├── index.html                  # 主页面
├── data/
│   ├── raw_data.json          # 原始数据
│   ├── kpi_results.json       # KPI计算结果
│   ├── statistics.json        # 统计分析结果
│   ├── top10_users.json       # Top10用户数据
│   └── anomalies.json         # 异常检测结果
├── css/
│   └── dashboard.css          # 样式文件
├── js/
│   ├── main.js                # 主逻辑
│   ├── charts.js              # 图表配置
│   └── interactions.js        # 交互逻辑
└── assets/
    └── images/                # 图标等资源
```

## 📱 响应式设计

### 桌面端 (>1200px)
- 3-4列卡片布局
- 完整显示所有图表
- 侧边栏导航

### 平板端 (768px-1200px)
- 2列卡片布局
- 图表适当缩小
- 顶部导航

### 移动端 (<768px)
- 单列垂直布局
- 卡片可折叠
- 底部导航栏
- 表格横向滚动
