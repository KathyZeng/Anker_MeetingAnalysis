## 📖 项目简介

会议改善效果评估系统是一个基于 **Streamlit** 的 Web 应用，用于分析飞书会议数据，评估会议改善措施的效果。

### 核心功能

- 📊 **KPI 指标分析**：日人均会议数、时长、即时会议占比
- 📈 **趋势可视化**：多周期会议数据对比和趋势图
- 👥 **人员分析**：Top用户、用户分层、个人详情
- 🔄 **数据管理**：CSV 上传、周期配置、自动保存
- 📄 **原始数据查看**：筛选、搜索、导出

## 🚀 快速开始

### 1. 环境要求

```bash
Python 3.8+
```

### 2. 安装依赖

```bash
pip install -r requirements.txt
```

### 3. 启动应用

**方式一：使用启动脚本（推荐）**
```bash
./start.sh
```

**方式二：直接启动**
```bash
streamlit run app.py
```

### 4. 访问应用

浏览器访问: http://localhost:8501

## 📋 使用流程

### 第一次使用

1. **上传数据** → 进入"📁 数据管理"页面，上传 CSV 文件
2. **选择周期** → 选择基线期和当前期，点击"加载数据"
3. **查看结果** → 切换到其他页面查看分析结果

### 后续使用

应用会**自动保存并加载上次的配置**，无需重新选择周期！

## 📊 核心指标

| 指标 | 计算方式 | 含义 |
|------|----------|------|
| 日人均会议数 | 平均值 | 平均每人每天参加的会议次数 |
| 日人均会议时长 | 平均值 | 平均每人每天的会议时长（分钟） |
| 即时会议占比 | (即时会议数 ÷ 总会议数) × 100% | 即时会议所占比例 |

## ⚠️ 常见问题

### Q1: 启动失败，提示端口被占用

```bash
lsof -ti:8501 | xargs kill -9
streamlit run app.py
```

### Q2: KPI 显示为 0

进入"数据管理"页面，选择周期并点击"加载数据"
